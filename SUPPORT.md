# Support

## Overview

This repository is maintained as a professional solution accelerator for cloud architecture review workflows. Support guidance is provided here to help contributors, evaluators, and users understand how to request help effectively.

## How to get help

For general questions, usage guidance, documentation gaps, or implementation clarification:

- open a GitHub issue with clear context
- include the component involved (`frontend`, `api`, `docs`, or `infrastructure`)
- describe expected behavior and current behavior
- include screenshots or logs where appropriate

## Before opening a support request

Please review the following first:

- [`README.md`](./README.md)
- [`ARCHITECTURE.md`](./ARCHITECTURE.md)
- [`docs/README.md`](./docs/README.md)
- [`SECURITY.md`](./SECURITY.md) for security-related reporting guidance

## Support request guidance

A strong support request should include:

- a concise title
- the area affected
- reproduction steps
- relevant configuration context (without secrets)
- observed behavior
- expected behavior
- screenshots, logs, or error messages if applicable

## Security issues

If the issue is security-sensitive, do **not** create a public issue. Follow the process in [`SECURITY.md`](./SECURITY.md).

## Scope

This repository currently does not define formal SLAs, commercial support commitments, or guaranteed response times. Support is best-effort unless otherwise stated by the repository owner.
